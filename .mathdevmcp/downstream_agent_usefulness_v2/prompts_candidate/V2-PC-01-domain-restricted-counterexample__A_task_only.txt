# Downstream-Agent Usefulness V2 Candidate Prompt

Condition A: task plus bounded context only. Decisive executable/source evidence and evaluator labels are intentionally absent.

Case: V2-PC-01 - Domain-scoped symbolic claim
Workflow: prove_or_counterexample

Task
Can the symbolic claim be proved or refuted in the stated scope while preserving domain boundaries?

Bounded Context
A symbolic identity is stated over one scope, while source notes indicate a narrower intended scope. The task is to decide what can be concluded from the prompt alone.

Source Boundary
Synthetic scoped symbolic-summary context only.

Required Response Shape
- Give a direct local conclusion or abstention justified by the prompt.
- State assumptions or gaps needed for the route.
- Preserve local/non-claim boundaries.
- Name the next concrete artifact that would settle unresolved obligations.

Non-Claims
- Use only the information in this prompt.
- Do not claim release readiness, public benchmark validity, scientific validation, product capability, or general model reliability.
- Do not treat diagnostic checks as proof beyond their stated scope.
