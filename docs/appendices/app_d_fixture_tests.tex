\section{Seeded mismatch fixture tests}

A particularly important implementation test class is the seeded mismatch fixture. In these tests, a small LaTeX fragment and a small code fragment intentionally disagree about a required mathematical term, algorithm component, dimension, or correction term. The consistency layer should then report a mismatch and identify the missing terms or relevant location. These fixtures are not substitutes for real-project evaluation, but they are an effective regression guard while the platform is still young.
