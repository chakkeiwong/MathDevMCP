# MathDevMCP Benchmark Read Me First

## Purpose

This short guide tells future readers where to start when navigating the
benchmark note stack.

The benchmark now has enough policy, execution, calibration, and checkpoint
artifacts that reading notes in arbitrary order is likely to create confusion.

Use this guide to separate:

- governing source-of-truth documents,
- current synthesis documents,
- and historical evidence documents.

## 5. Practical reading order

If you want the fastest useful understanding of the benchmark, read in this
order:

1. `mathdevmcp-real-tasks-benchmark-master-program-2026-06-17.md`
2. `mathdevmcp-benchmark-note-taxonomy-2026-06-19.md`
3. `mathdevmcp-real-tasks-benchmark-status-dashboard-2026-06-19.md`
4. `mathdevmcp-benchmark-current-state-assessment-2026-06-19.md`
5. `mathdevmcp-benchmark-acceptance-assessment-2026-06-19.md`
6. then only the historical notes relevant to the question you are asking

## Non-claim boundary

This guide does **not** mean every note listed under “historical evidence” is
obsolete or irrelevant.

It means those notes should usually be read as supporting evidence, while the
current synthesis layer should be treated as the best current benchmark state.
