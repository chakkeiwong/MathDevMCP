# Boehl Blind Gap-Closure Master Program

Date: 2026-07-22

Status: reviewed for execution

## Mission

Improve MathDevMCP as an exploratory, high-standard, rigorous, agent-facing
applied-mathematics audit system. The repair targets general economics,
finance, marketing, management, and related social-science documents. The
Boehl papers are a diagnostic case, not the definition of the product.

Search broadly for mathematical, statistical, accounting, timing, and source-
boundary failures; remain strict at the claim boundary. A candidate may be
speculative, but a `confirmed_defect` must have a closed target, explicit
assumptions, source anchors, and deterministic evidence.

Observed gaps:

1. Level-to-linearized, SDF, timing, ownership, and coefficient relationships
   remain obligation-level tensions instead of executable comparisons.
2. Candidate edge generation based on shared words produces loose pairings.
3. C.75-style normalization/sign/timing language is not robust to PDF text
   extraction variants.
4. C.77-style bank-held versus aggregate ownership is not represented as a
   typed domain comparison.
5. C.79-style coefficient/sign differences are not formalized into a bounded
   symbolic target.
6. ResearchAssistant parser fallback does not discover or ingest adjacent
   authoritative source packages, YAML, code, appendices, or errata.
7. Six obligation families remain `not_checkable` when source structure or
   code/data is absent; the report does not yet distinguish missing evidence
   from failed checking at enough granularity.
8. PDF equation candidates retain page/line context but not reliable visual
   crops or equation segmentation.

## Research Intent Ledger

| Field | Contract |
| --- | --- |
| Main question | Can typed equation roles, conservative semantic relations, and safe bounded formalization convert a useful subset of equation-pair tensions into reproducible checks without increasing false defects? |
| Candidate mechanism | Role extraction, domain/timing/coefficient metadata, explicit relation candidates, safe SymPy formalization, and source-package discovery. |
| Expected failure modes | Overfitting to Boehl labels, parser glyph corruption, false semantic links, hidden assumptions, unsafe symbolic execution, provider drift, and answer-key leakage. |
| Primary promotion criterion | On the predeclared generic fixture matrix: 100% of non-abstaining closed cases are classified correctly, at least 80% of closed cases are non-abstaining, 100% of ambiguous cases abstain, zero false `confirmed_defect` promotions occur, and every finding has a replayable evidence chain. The same-paper replay is descriptive only. |
| Promotion veto | A source-specific rule, hidden assumption, unbound symbolic variable, parser disagreement treated as truth, arbitrary execution, missing evidence chain, or compact/detail divergence. |
| Continuation veto | Corrupted artifact, schema failure, source digest drift, or inability to distinguish an unsupported candidate from a checked target. |
| Repair trigger | Any false pair, missed generic fixture, unrecorded abstention reason, or tuned-replay result that worsens confirmed-defect precision. |
| Must not conclude | General recall, complete PDF understanding, code equivalence, causal validity, posterior validity, or correctness of the Boehl model. |

## Evidence Contract

* Baseline comparator: the frozen fresh blind artifact, its exact CLI command,
  input/provider/artifact digests, and its post-hoc issue comparison. The old
  implementation/environment snapshot is incomplete, so all same-paper
  baseline/replay differences, including tool-artifact fields, are descriptive
  and never promotion evidence.
* Primary metrics: a fixture confusion matrix with denominators for closed
  positive, closed negative, and ambiguous cases. All non-abstaining closed
  classifications must be correct; non-abstention coverage over all closed
  cases must be at least 80%; all ambiguous cases must abstain; false
  `confirmed_defect` promotions must be zero; 100% of findings must have source
  packet and check references. Candidate false-link rate is false inferred
  pairs divided by all inferred pairs and must be zero on the frozen negative/
  ambiguous subset. Same-paper exact/partial/missed results are descriptive.
* Hard veto diagnostics: digest mismatch, hidden provider failure, a confirmed
  defect without a closed target and assumptions, unsafe command execution,
  missing role/domain provenance, or a changed blind input.
* Descriptive diagnostics: finding count, candidate-edge count, parser count,
  payload size, runtime, and number of not-checkable obligations.
* Artifact: phase records, validator traces, fixture scorecard, second tuned-
  replay report/manifest/artifact, and final comparison.

## External-Tool-First Route

Before native formalization, the implementation must consider:

* ResearchAssistant for source/PDF/page and structured-source intake;
* SymPy for explicitly bounded algebra/calculus targets;
* DynareMCP only for supplied `.mod` inputs and fixed source diagnostics.

If a provider or backend is unavailable, the result is visible
`backend_abstention` or `not_checkable`; absence is not a scientific refutation.
No validator executes arbitrary user code or shell commands.

## Skeptical Plan Audit

This plan was audited before execution:

* A higher Boehl score is not the promotion criterion; generic fixture validity
  and evidence integrity come first.
* The second tuned replay is performed in a fresh session after implementation
  freeze. This session may use the prior blind report for repair, but the fresh
  agent may not see any answer key or comparison.
* No C-number, paper-specific equation label, or exact Boehl phrase is encoded
  as a rule. Tests use synthetic level/linearization, SDF, ownership, and
  coefficient fixtures with varied notation.
* Safe formalization requires an explicit expression, variable/domain record,
  and bounded operation. Otherwise it abstains.
* A candidate relation can remain `supported_tension`; it cannot become a
  defect merely because it is selected or symbolically parseable.
* A relation-derived `confirmed_defect` additionally requires explicit source
  evidence that the two authenticated expressions are asserted to be related.
  PDF parser candidates and page crops remain `supported_tension`; a crop
  localizes evidence but does not authenticate the expression fed to a backend.
  Promotion requires structured-source identity or an independently verified
  transcription with its own reviewer/provenance record.
* Parser improvements and bounding-box page regions are evidence localization,
  not faithful mathematical transcription. Promotion requires structured
  source identity or an independently verified transcription record.
* The second run on the same PDFs is an **instruction-compliant tuned replay**,
  not a product holdout: implementation used the prior report and issue
  classes, while comparator non-access is self-reported rather than enforced.
  Generalization requires a separately sealed cross-domain corpus.
* Baseline and replay tool/agent settings are not fully reproducible from the
  prior manifest. All same-paper comparisons are descriptive and never
  promotion evidence. The new run records code-file digests, Python/package
  versions, environment controls, provider identity, prompt, and input/output
  digests for future reproducibility.
* All fixtures and answer records, including notation-renaming, distant-
  reference, adversarial-distractor, and cross-domain cases, freeze in Phase 0.
  There are at least 12 cases: four closed-positive, four closed-negative, and
  four ambiguous cases across at least three domains and every planned
  relation family. They are visible engineering regressions, not sealed
  holdouts or product-generalization evidence.
* One implementation program and at most two focused repair loops are allowed
  before the replay. Opening replay comparison answers ends this one-shot
  descriptive replay protocol.
* The visible fixture matrix directly exercises the bounded formalizer;
  orchestrator packet binding, PDF fail-closed promotion, source discovery,
  and artifact identity are separate end-to-end regressions. Matrix metrics
  must not be reported as relation-matcher or paper-level recall.

## Phases

### Phase 0: Baseline Freeze And Repair Fixtures

Objective: freeze the baseline and all synthetic, domain-neutral fixtures that
expose each root cause, including renaming, distant-reference, adversarial-
distractor, and cross-domain cases.

Entry: fresh blind artifacts and current focused tests are readable.

Artifacts: baseline manifest, complete fixture corpus, fixture answer records, Phase 00
subplan/result.

Checks: source digests; at least 4 closed-positive, 4 closed-negative, and 4
ambiguous cases; at least three domains and every relation family; no answer-
key content in fixtures.

Handoff: every planned repair has at least one positive, negative, and
ambiguous fixture.

Stop: baseline or fixture isolation cannot be verified.

### Phase 1: Source Authentication And Page Objects

Objective: add bounded page/equation crop references and structured-source
identity where available; explicitly classify parser-only expressions as
unauthenticated.

Entry: Phase 0 frozen.

Artifacts: page bounding boxes/crop references, structured-source binding,
authentication status, visual-source fixtures, Phase 01 result.

Checks: page bounds, label association, crop digest, source digest, parser
disagreement, missing-renderer abstention, and no crop-to-LaTeX certification.

Forbidden: backend promotion from unauthenticated parser expressions.

Handoff: every mathematical packet is `source_authenticated`,
`page_region_located`, `independently_verified_transcription`, or
`parser_candidate_only`. Only the first and third states can authorize a
relation-derived mathematical promotion.

An `independently_verified_transcription` requires a separate decision artifact
with reviewer identity distinct from the extractor/implementer, source and page
region digests, canonical transcription digest, `agree|disagree|abstain`
decision, and conflict notes. Only `agree` with matching digests authorizes the
state; disagreement or missing identity forces `parser_candidate_only`.

Stop: authentication state is missing or a crop/source digest is unbound.

### Phase 2: Typed Equation Roles And Source Context

Objective: extend packets with equation role, object/domain, timing, variable,
coefficient, and normalization candidates while retaining raw evidence.

Entry: Phase 1 authentication states validate.

Artifacts: versioned IR extension, role extractor, context-window tests,
parser-variant fixtures, Phase 02 result.

Checks: varied notation, PDF line wrapping, equation labels, context limits,
role confidence, unresolved-role preservation.

Forbidden: interpreting parser glyphs as certified symbols or inferring a role
without confidence and source text.

Handoff: validators can consume typed candidates and distinguish explicit,
inferred, and unresolved metadata.

Stop: role metadata cannot retain raw source or confidence.

### Phase 3: Conservative Relation Matching

Objective: replace broad shared-word pairing with relation candidates requiring
role compatibility, symbol overlap, object-domain compatibility, and proximity.

Entry: Phase 2 typed candidates validate.

Artifacts: relation matcher, match score/explanation, false-link fixtures,
ambiguity records, Phase 03 result.

Checks: varied-domain positive/negative/ambiguous pairs, duplicate suppression,
threshold boundary, relation evidence chains, and source-specific-rule scan.

Forbidden: adjacency or common words establishing dependency; choosing a pair
only because it improves the Boehl score.

Handoff: loose candidate false-link rate is measured and ambiguous pairs
abstain with a reason.

Stop: matcher produces a promoted defect from an ambiguous pair.

### Phase 4: Source Discovery And Evidence Closure

Objective: discover adjacent LaTeX/YAML/code/data/appendix/errata candidates,
record provenance, and make missing-evidence reasons structured before
relation-derived formalization can be promoted.

Entry: Phase 3 relations exist.

Artifacts: source-package manifest adapter, provider capability records,
evidence-closure statuses, discovery tests, Phase 04 result.

Checks: local source package, absent package, provider failure, digest changes,
candidate authority versus unverified candidate, path confinement, and all
obligation dispositions.

Forbidden: treating a candidate file as authoritative without identity or
executing discovered code automatically.

Handoff: every unresolved obligation names the exact missing evidence or failed
provider route; authenticated evidence is available before promotion.

Stop: source/provider failure is hidden or authority is inferred silently.

### Phase 5: Safe Formalization And Generic Validators

Objective: execute bounded checks for normalization/sign/timing, level-to-
linearized movement, ownership domains, coefficient identity, dimensions, and
closed arithmetic using external deterministic tools where applicable.

Entry: Phase 4 source authority/closure records exist.

Artifacts: formalization protocol, SymPy adapter, validator traces, paired
fixtures, Phase 05 result.

Checks: use an existing reviewed bounded adapter where its exact operation
matches; otherwise abstain. Any new adapter must enforce an allowlisted
Python-expression AST (`Name`, numeric constants, `+ - * / **`, unary signs
only), maximum 1,000 characters, 256 AST nodes, nesting depth 32, 64 symbols,
20-digit integer literals, exponent magnitude 12, and bounded operation/
polynomial complexity; an identifier allowlist;
CPU, address-space, stack, wall-time, and output limits; isolated fixed worker
code with no network/file operations; symbolic equality/difference, derivative
under declared variables, branches/poles/nonzero denominators and unsupported
piecewise/complex cases, positive/zero-domain cases, lag/lead identity,
ownership-set comparison,
coefficient-name mismatch, tolerance handling, malformed-expression abstention,
and no arbitrary execution. Unresolved branches, poles, denominators, domains,
or conditioning assumptions force abstention.

Evidence: `confirmed_defect` requires a closed target, assumptions/domain,
backend/version, structured-source or independently verified-transcription
packets, and an explicitly anchored
source assertion that the objects should be related. A coefficient-name
difference alone is a diagnostic. Otherwise return `supported_tension` or
`not_checkable` with a reason code.

Handoff: generic fixtures identify intended defects and abstain on ambiguous
ones without false promotion.

Stop: any formalization silently adds assumptions or treats parser output as
certified LaTeX.

### Phase 6: Contract, Benchmark, And Replay Preparation

Objective: wire findings, traces, paging, docs, and frozen benchmark scoring;
review the exact fresh-agent prompt and verify that it excludes the answer key.

Entry: Phases 1--5 pass focused checks.

Artifacts: updated public contract, docs, fixture confusion matrix, replay
prompt, allowed-file manifest, environment/tool manifest, access-attempt log
template, and Phase 06 result.

Checks: compact/detail parity, MCP stable/all surfaces, artifact hashes,
predeclared frozen Phase 0 fixture thresholds, source-specific rule scan, and
prompt/allowed-file review.

Forbidden: running the tuned-replay agent before the prompt and artifact paths
are frozen.

Handoff: a fresh agent receives an explicit allowed-file manifest containing
only the two PDFs and public CLI/package snapshot. Filesystem isolation is
instructional rather than an OS security boundary, access non-use is
self-reported, leakage prevention is unverified protocol evidence, and no
holdout claim depends on it.

Stop: leakage, schema mismatch, or fixture regression.

### Phase 7: Instruction-Compliant Tuned Replay And Post-Hoc Comparison

Objective: run the identical two-paper CLI test in a new Codex session and
compare only after the report/artifact are frozen. This tests fresh-user
discoverability after tuning, not out-of-sample system generalization.

Entry: Phase 06 close, implementation freeze, and prompt/manifest freeze.

Artifacts: fresh replay report, manifest, audit artifact, post-hoc comparison,
Phase 07 result.

Checks: exact input digests, self-reported comparator non-access, tool-only
versus agent explanation attribution, exact/partial/missed scoring, false-
positive review, and remaining-gap classification.

Forbidden: exposing answer-key content in the prompt, changing inputs, calling
the run blind or holdout-blind, making leakage-sensitive conclusions, or
rerunning after opening comparison answers.

Handoff: second-run evidence shows which gaps were closed, unchanged, or
regressed.

Stop: input drift, leakage, or missing frozen artifact.

### Phase 8: Required Independent Red-Team And Closeout

Objective: review code, tests, plan execution, blind protocol, and scientific
non-claims; produce the final gap report.

Entry: Phase 07 artifacts complete.

Artifacts: independent review, full relevant test summary, final result, reset
memo, and refreshed next plan if semantic gaps remain.

Checks: focused/broad relevant tests, MCP smokes, diff review, artifact hashes,
and required independent read-only Codex or Claude review.

Forbidden: claiming scientific correctness, general recall, or production
readiness from the second tuned replay case.

Handoff: final report states measured improvements and residual gaps.

Stop: public contract or evidence integrity remains defective.

## Automatic Phase Loop

At each phase close, run checks, write the phase result, draft/refresh and
review the next subplan, then launch it if ready. At most two focused repair
loops may be used before the one-shot same-paper replay. A test failure triggers
a bounded repair and focused rerun. A scientific low-recall result triggers a
future program; the replay is not adaptively rerun after comparison.
