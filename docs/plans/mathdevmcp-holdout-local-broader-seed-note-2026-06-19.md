# MathDevMCP Holdout-Local Broader Seed Note

## Date

2026-06-19

## Scope

This note records a further bounded broadening of the **local-only holdout
seed**.

It remains a local population and local fixture checkpoint, not holdout-backed
generalization evidence.

## What changed locally

The local holdout seed now spans four local-only example families:

1. `latex-papers` chapter-neighborhood retrieval/provenance
2. `dsge_hmc` blocker-preservation result-note family
3. `MacroFinance` blocked-diagnostic/result-note family
4. `MacroFinance/ResearchAssistant` inventory-structure discipline family

The local candidate-answer fixture set has also been extended so that every
currently populated local holdout case has a matching local candidate fixture.

## Why broadening was justified

This broadening improves local holdout breadth in two ways:

- it adds another source family not already represented in the local holdout
  seed;
- it adds a lighter inventory-discipline/evidence-boundary style case distinct
  from the existing heavier blocker and retrieval families.

This is still a tiny local seed, but it is more representative than the prior
three-family version.

## Current local family mix

The local-only holdout seed now includes:

- `retrieval_and_provenance`
- `evidence_boundary_discipline` from `dsge_hmc`
- `evidence_boundary_discipline` from `MacroFinance`
- `evidence_boundary_discipline` from `MacroFinance/ResearchAssistant`

The local candidate-answer fixtures now cover all currently populated local
holdout entries.

## What this still does **not** mean

This checkpoint still does **not** mean:

- the holdout-local tier is broad enough for strong generalization claims,
- the benchmark is complete,
- the holdout tier is ready for workflow/gate/release coupling,
- inventory-style cases are now public benchmark fixtures.

## Why this matters

The local holdout tier is now moving from a tiny multi-family seed toward a
slightly broader local evaluation seed. That is exactly the direction the master
program and strategic checkpoints recommended.
