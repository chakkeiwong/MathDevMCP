### Veto diagnostics

The parser veto is true for any:

- current compact/materialized/source reconstruction drift;
- missing, duplicated, unreviewed, or mismatched invocation/raw artifact;
- executable, version, argv, environment, timeout-value, source, output, log,
  stdout, or stderr binding mismatch;
- source mutation;
- unclassified capability state or status-precedence mismatch;
- limitation omitted, mismatched, or promoted;
- observation without independent raw provenance;
- valid independent missing-requested-label contradiction;
- frozen identity rewrite or unjustified specialist selection.

The parser veto is not set solely because a fully reconstructed specialist
record is `timed_out`, `nonzero_exit`, `malformed_output`, or
`valid_not_source_mappable`. Those states must have explicit limitations, empty
promotion, and no observations from invalid bytes. They remain explanatory and
block that specialist.

separate target-specific runtime hypothesis requires zero timed-out source calls;
the formal parser-fidelity test fails if any reviewed ceiling is reached, and
the resulting nonzero local action sets the ordinary artifact-integrity veto.

explanatory only.
