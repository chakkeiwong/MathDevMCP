## Mission

Close the engineering gaps exposed by the frozen D446/D447 capstone while
preserving MathDevMCP's governing boundary: exploratory in search and rigorous
at the claim boundary.

The primary target is resumable complete audit/rigor coverage for all 566
indexed non-nested D447 equation labels, separating 434 current canonical
adapter targets from 132 source-bound typed relation-shape abstentions.
Derivation-tree execution targets the 434 canonical targets and must carry the
132 abstentions in its coverage ledger rather than fabricate tree targets.
Secondary targets are packet latency,
first-class function-scope diagnostics, and a reviewed ownership-or-abstention
route for the seven lookup-only nested-alignment labels.

Independent generalization is not an engineering target in this program. It
remains `not_tested_no_verified_clean_holdout` until a provenance-clean source
is supplied or independently selected before result unblinding.

## Research Intent Ledger

| Field | Predeclared intent |
| --- | --- |
| Main engineering question | Can deep document workflows preserve exact evidence and completed work across bounded batches, crashes, and restarts, then assemble the same semantic result as an uninterrupted run? |
| Candidate mechanism | Immutable document sessions, content-addressed per-label or per-target checkpoints, deterministic aggregation, and validated evidence reuse. |
| Exact comparator | Existing uninterrupted workflow on synthetic fixtures and the 18-label C.71--C.77 slice; the 64-label rung is a scale diagnostic before the 566-label capstone. |
| Expected failure mode | Repeated full-directory parsing/parser-policy work, all-or-nothing synthesis, no durable per-target progress, stale checkpoint acceptance, order-dependent aggregation, or concurrency multiplying memory/backend initialization. |
| Primary promotion criterion | Interrupted-and-resumed and uninterrupted executions have equivalent canonical semantic payloads; all 566 audit/rigor labels are accounted for once as 434 canonical targets plus 132 typed abstentions; stale/corrupt/cross-session records fail closed; the tree completes all 434 executable targets and retains the 132-label non-executable ledger. |
| Promotion veto | Source drift, label omission/duplication, obligation mismatch, unsafe record reuse, aggregation semantic drift, diagnostic-to-proof promotion, branch erasure, or publication enablement. |
| Continuation veto | Frozen source drift, corrupt required baseline, repeated unhandled artifact-store failure, or demonstrated inability to aggregate without changing workflow semantics. |
| Repair trigger | A localized implementation, schema, validation, or performance defect with a focused regression route. |
| Explanatory diagnostics | Per-label and per-batch runtime, cache hits, bytes, memory, batch counts, and total wall time. |
| Must not be concluded | D447/BGS correctness, theorem proof, source-author error, implementation equivalence, publication readiness, release readiness, or independent generalization. |

## Evidence Contract

Before every scale run, preserve:

- exact source file and digest;
- ordered requested-label inventory;
- extraction obligation IDs/digests;
- workflow options and schema versions;
- exact baseline or prior rung;
- batch command, environment, elapsed time, and output paths;
- hard-veto results separately from explanatory performance metrics.

Primary pass conditions:

1. every requested label has exactly one validated record or one explicit typed
   ownership/unsupported status;
2. no record from a different session, source, configuration, label, or
   obligation can be reused;
3. resuming after interruption does not recompute validated completed records;
4. aggregation restores canonical requested order;
5. small and 18-label aggregate semantics match the uninterrupted comparator;
6. publication remains disabled and backend/CAS results remain diagnostic
   unless separately certified.

Runtime is an engineering boundary, not evidence of mathematical correctness.

## Skeptical Pre-Execution Audit

| Risk | Finding and required repair |
| --- | --- |

| Proxy promotion | Runtime/cache-hit improvements cannot pass correctness gates. Canonical identity, coverage, equivalence, and claim boundaries are primary. |

## Phase 0: Baseline And Contract Fixtures

Objective: freeze semantic comparison projections and create mutation fixtures.

Required artifacts:

- small synthetic label set;
- frozen 18-label D447 set;
- frozen 64-label D447 scale rung selected before timing results;
- canonical semantic projections that exclude explanatory runtime, artifact
  path, and scheduling fields;
- mutation matrix for source, label order, obligation, configuration, schema,
  corrupt bytes, and duplicate/missing records.

Checks:

- existing uninterrupted workflows still pass focused tests;
- projections retain all claim, evidence, blocker, proposal, coverage, and
  publication-boundary fields;
- excluded fields are explicitly enumerated and explanatory only.

Handoff: Phase 1 starts only when the equivalence comparator is meaningful.

## Phase 1: Immutable Document Audit Session

Objective: build one validated session shared by label-audit consumers.

Required session fields:

- schema version and session ID;
- root and exact target file;
- source digest;
- ordered requested labels;
- obligation IDs/digests or typed lookup-only status;
- parser/index schema identities;
- backend, paragraph-context, summary, and parser-policy options;
- backend availability snapshot with explanatory runtime removed from durable
  identity.

Checks:

- deterministic session identity;
- source drift and option drift fail closed;
- wrong file or reordered labels produce a different session;
- unchanged session reload succeeds without source ambiguity.

Forbidden: a session record is not an audit result or proof.

## Phase 2: Durable Per-Label Audit Records

Objective: produce atomic, immutable `proof_audit_v2_result` records.

Implementation requirements:

- reuse a session-bound index/cache and parser-policy input;
- validate result label, source digest, obligation identity, contract, and
  options before publication;
- publish canonical bytes with no replacement;
- expose pending/completed/failed counts;
- allow a bounded `max_new_records` batch;
- skip only fully validated completed records on restart;
- distinguish candidate audit status from record-production failure.

Tests:

- interruption after arbitrary record counts;
- restart avoids recomputation;
- corrupt, partial, symlinked, stale, duplicate, and cross-session records fail;
- sequential output order is independent of completion order;
- one typed label failure does not invalidate unrelated completed records.

## Phase 3: Deterministic Audit/Fix Aggregation

Objective: run the existing document synthesis over complete precomputed audits.

Required behavior:

- exact coverage validation before synthesis;
- no label audit recomputation;
- existing proposal builder, validation, nonclaims, and high-level envelope are
  reused;
- tool ledgers identify reused evidence without hiding original backend uses;
- aggregate detailed artifact remains resolvable when compact previews truncate.

Equivalence gates:

- exact small-fixture semantic projection;
- exact 18-label semantic projection;
- order, duplicate, missing, and stale mutations fail;
- 64-label interrupted/resumed run completes within bounded batches.

## Phase 4: Rigor Consumption

Objective: allow rigor to consume the complete aggregate audit/fix artifact.

Checks:

- existing exact filename, digest, label-set, and complete-coverage checks remain;
- session and aggregate identities are validated;
- rigor performs no label-audit recomputation;
- small/18-label rigor semantic projections match the comparator;
- compact result remains bounded and the detailed result is resolvable.

## Phase 5: Durable Per-Target Derivation Trees

Objective: checkpoint each tree target and reuse the existing document assembler.

The tree session covers the 434 currently canonical adapter targets. The 132
source-bound typed relation-shape abstentions remain in the full-document
coverage ledger and are not eligible for fabricated semantic packets.

Required target identity:

- source/session identity;
- label, row, obligation ID/digest;
- budget profile, maximum attempts, backend environment, search mode, grounding
  policy, relevant capability snapshot, and schema versions.

Required behavior:

- prepare shared sections, rows, displays, index, capabilities, and integrations
  once per bounded invocation;
- persist each target result atomically;
- resume without recomputing validated targets;
- use the same coverage, integrity, blocker, compiler, tool-use, and publication
  assembler as the uninterrupted path.

Equivalence gates mirror Phase 3. Publication must remain disabled even with
complete target coverage.

## Phase 6: Packet Reuse And Latency

Objective: build proof and negative-evidence packets from validated session
records.

Checks:

- cached and uncached semantic projections match;
- wrong label/source/obligation/configuration fails closed;
- packet artifact and compact transport contracts remain unchanged;
- representative D447 packet calls do not rebuild the full document index or
  parser policy.

Performance is reported descriptively. The correctness pass is semantic
equivalence plus exact reuse validation.

## Phase 7: First-Class Scope Diagnostic

Objective: replace ambiguous `not_triggered` with explicit static states:

- `not_applicable`;
- `scope_covered_structurally`;
- `scope_limited`;
- `scope_unverifiable_no_callable_boundary`.

The D447 assignment-fragment example should be classified as lacking a callable
function boundary, while retaining the localized missing-`theta` mismatch.

Forbidden: static signature evidence is not semantic implementation proof.

## Phase 8: Nested-Alignment Ownership Or Abstention

Objective: determine whether any of the seven lookup-only labels can receive
unambiguous mathematical row/display ownership.

External-tool-first route:

- current byte-preserving locator as baseline;
- LaTeXML 0.8.6 as an available diagnostic comparison;
- SymPy/Sage are inapplicable to source ownership;
- Lean is relevant only after ownership and formalization, not for TeX row
  localization.

Promotion condition:

- one exact, documented ownership rule;
- byte/source-span evidence;
- no sibling-branch collapse;
- synthetic and all-seven D447 regressions.

Otherwise retain `nested_display_ownership_required` as the correct typed
abstention. Backend disagreement or absence is diagnostic, not refutation.

## Phase 9: Calibration Ladder

Entry repair discovered during the first 1/4 rung: the old 566-label inventory
contains 132 exact source-bound labels whose relation shape is quarantined and
which therefore emit no adapter target. The plan was repaired before further
scale execution. Audit sessions bind both canonical and typed-abstention
obligations; tree sessions bind canonical targets only and report exclusions.

Run serial batches with candidate sizes `1`, `4`, `8`, and `16`.

Stop increasing batch size when:

- any batch exceeds ten minutes;
- memory growth is unbounded;
- an artifact/identity veto fires;
- restart or semantic equivalence fails.

Select the largest size whose observed upper runtime is below eight minutes.
Observed timings are descriptive because they are not replicated statistical
performance comparisons.

Concurrency may be tested only after serial correctness passes. It cannot
become the default without deterministic-equivalence, memory, and backend
isolation evidence.

## Automatic Repair Loop

At each phase boundary:

1. run focused and adjacent checks;
2. write or refresh the phase result;
3. inspect failed gates and classify implementation, harness, backend, scale,
   source, or scientific uncertainty;
4. repair localized defects and rerun the smallest affected comparator;
5. continue automatically when entry conditions pass;
6. stop only for a true continuation veto or a material human boundary.

## Global Non-Claims

- Complete workflow coverage is not proof of D447 or BGS mathematics.
- CAS simplification, parser agreement, source localization, and generated
  branches are not theorem certificates.
- A resumable artifact is local byte-identity evidence, not publication or
  scientific authority.
- No result here establishes independent generalization, product superiority,
  release readiness, or a new default policy.
